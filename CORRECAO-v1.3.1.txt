CORREÇÃO v1.3.1 - 2026-02-04

Problema: Foreign Key Violation ao salvar exames
Erro: PostgresException 23503 - FK_exames_tipos_exame_tipo_exame_id

Causa Raiz:
  GetOrCreateTipoExameAsync criava TipoExame mas NÃO salvava no banco.
  Apenas adicionava ao contexto: _context.TiposExame.Add(novoTipo)
  
  Depois, ao tentar salvar todos os Exames juntos, as foreign keys
  dos TipoExames ainda não existiam no PostgreSQL.

Arquivo corrigido:
  src/ExamAI.Infrastructure/Repositories/ExamRepository.cs

Mudança aplicada:
  Linha 311: Adicionado await _context.SaveChangesAsync(cancellationToken);
  
  ANTES:
    _context.TiposExame.Add(novoTipo);
    return novoTipo; // ❌ Não salvou!
  
  DEPOIS:
    _context.TiposExame.Add(novoTipo);
    await _context.SaveChangesAsync(cancellationToken); // ✅ Salva AGORA!
    return novoTipo;

Impacto:
  - TipoExame agora tem ID válido ANTES de criar Exame
  - Foreign key sempre válida
  - Salvamento de exames funciona completamente

Build: ✅ OK
